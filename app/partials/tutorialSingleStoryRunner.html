BDD Assistant Tutorial

Single Story runner

Single Story Runner is commonly used during the specification and development of some story. After scenarios are written, developers should use the BDD in the TDD fashion:
  • Run scenarios and confirm that at least one of the failed or has pending steps
  • Implement the first one that failed or has pending steps
  • Run scenarios and confirm that the previous failed or with pending steps is working
  • Refactor
  • Repeat

Run Story

Once inside an story, it can be run by clicking the "Run Story" button.

Run Story parameters dialog

"Run Story" action brings the story runner dialog that can be used to set required parameters. For example, browser type and size can be specified in the story runner dialog. To proceed and run the specified story using parameters set in this dialog, click the "Run Story" button.

Run Story outcome

Run Story execution status

Once the story run starts, progress bar will appear in the top right corner of the screen. After the story run is finished, progress bar changes its status to successful or failed depending on the outcome.

Pending steps

If story that was run contains pending steps, they will be listed inside the story run outcome section. To create the implementation of an pending step, please click on the corresponding link.

Reports

Detailed report of the execution of all scenarios is provided as well. If there are pending or failed steps, they are marked appropriately.
