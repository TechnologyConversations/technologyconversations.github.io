BDD Assistant Tutorial

Story Form

Story Form is a representation of an BDD story. It is divided into 3 parts: Name, Story and Scenarios.

Name

Name, together with the directory path, forms the unique identification of the BDD story.

Story

Story section is divided into 5 sections. None of the sections is mandatory.

Description

This free-text field is for informational purposes only. It can be used as a way to provide any type of information not already covered by other sections (i.e. narrative, scenarios, etc). In cases where narrative is not appropriate, description can be used as an alternative.

Narrative

Narrative is equivalent of a user story. The purpose of narrative is to capture initial idea or requirement and initiate conversation that will result in one or more scenarios. It consists of 3 fields:

  • In order to [ACCOMPLISH SOMETHING]
  • As a [USER OR ROLE THAT NEEDS THE NEW FEATURE]
  • I want to [DO SOMETHING]

Lifecycle

Lifecycle is split into two sub-sections: Before and After. Each can define a set of steps that are run before or after each scenario. Lifecycle is very useful when all scenarios in one BDD story start or end with the same set of steps.

Scenarios

Each BDD story has at least one scenario. Each of the scenarios is separated into 3 sections: Meta, Steps and Examples.
