What is Behavior Driven Development?

BDD is a second-generation, outside-in, pull-based, multiple-stakeholder, multiple-scale, high-automation, agile methodology. It describes a cycle of interactions with well-defined outputs, resulting in the delivery of working, tested software that matters.

BDD was created by Dan North as as a response to the issues encountered teaching test-driven development (TDD) and is basically a way to rethink the approach to unit testing and acceptance testing.

BDD Assistant

BDD Assistant is an open source project that pretends to facilitate the creation and execution of Behavior Driven Development (BDD) stories as a way to better define the requirements. Latest release can be downloaded from TechnologyConversationsBdd releases.
